Генерация содержимого

В некоторых случаях CSS позволяет отображать содержание, которого нет в дереве документа:

  • генерация содержимого с помощью псевдоэлементов :before и :after
  • автоматическая нумерация с использованием счетчиков
  • создание маркеров и отображение списков

Задание содержимого: свойство content

content с псевдоэлементами :before и :after для генерации

  • "строка" — текст
  • url(...) — содержимое ресурса (напр., картинка)
  • counter(имя,стиль) — значение счетчика
  • attr(...) — значение атрибута элемента
  • open-quote, close-quote — кавычка, определяемая quotes
  • no-open-quote, no-close-quote — ничего, но счетчик вложения кавычек изменяется на 1

Кавычки: свойство quotes

Свойство quotes задает пары символов кавычек, возможно для нескольких уровней вложенности

  • 0022 " двойная кавычка
  • 0027 ' одинарная кавычка (апостроф)
  • 2039 ‹ открывающая одинарная угловая кавычка
  • 203A › закрывающая одинарная угловая кавычка
  • 00AB « открывающая двойная угловая кавычка
  • 00BB » закрывающая двойная угловая кавычка

Кавычки: свойство quotes

  • 2018 ‘ открывающая одинарная кавычка
  • 2019 ’ закрывающая одинарная кавычка
  • 201A ‚ закрывающая нижняя одинарная кавычка
  • 201C “ открывающая двойная кавычка
  • 201D ” закрывающая двойная кавычка
  • 201E „ закрывающая нижняя двойная кавычка

Кавычки: свойство quotes

		Q { quotes: "\00AB" "\00BB" "\201C" "\201D" }

Автоматическая нумерация

Счетчик — это идентификатор, которому присваиваются целые значения с помощью counter-reset и counter-increment

Текущее значение счетчика может использоваться при генерации содержимого свойством content:counter(имя,стиль)

Элементы с display:none не изменяют значения счетчиков

Автоматическая нумерация

		body { counter-reset: chapter -1}
		h1:before {
			counter-increment: chapter;  /* Увеличить chapter на 1 */
			counter-reset: section;      /* Обнулить section */
			content: "Глава " counter(chapter,upper-roman) ". ";
		}
		h2:before {
			counter-increment: section;  /* Увеличить section на 1 */
			content: counter(chapter) "." counter(section) " ";
		}

Маркеры

display:marker в :before и :after — однострочный прямоугольник вне главного прямоугольника элемента

Создается только когда свойство content генерирует содержание

Прямоугольник маркера имеет рамку и заполнитель, не имеет границ

marker-offset — расстояние между ближайшими краями рамок маркера и главного прямоугольника по горизонтали

Маркеры обычно бывают у списков

Элементы списков создаются присваиванием display:list-item

Вид номера — list-style-type: armenian | decimal | georgian | lower-alpha | lower-greek | lower-latin | lower-roman | upper-alpha | upper-latin | upper-roman

Или картинка — list-style-image:url(...)

Позиция — list-style-position: inside | outside

Всё вместе — <ul style="list-style: square inside">

Страничные устройства вывода

Страничные устройства вывода

Синтаксис @page [селектор-страницы] { правила; ... }

Селектор-страницы — это псевдоклассы :first, :left, :right или имя:

		@page rotated { size: landscape }
		table { page: rotated; ... }

Размер страницы

Свойство size: размеры | auto | portait | landscape

Для задания границ прямоугольника страницы — margin

Управление разрывами страниц

Разрыв страницы перед элементом: свойство
page-break-before: auto|always|avoid|left|right

После элемента: свойство page-break-after

Разрыв страницы внутри элемента: свойство page-break-inside: auto | avoid

Количество строк в конце страницы: свойство orphans:число

Количество строк в начале страницы: свойство widows:число

БЭМ от Яндекса

		.block_element {...}
		.block_element-modifier {...}

Хорошо: изолированность модулей друг от друга и отсутствие непредсказуемых каскадов селекторов

Плохо: не всегда длинные названия классов удобны для использования (для крупных проектов есть смысл использовать инструменты, автоматизирующие работу с БЭМ-разметкой)

Scalable and Modular Architecture for CSS

smacss.com

Scalable and Modular Architecture for CSS

  • Base rules — стили основных элементов сайта: body, input, button, ul, ol и т.п., в этой секции используются в основном селекторы тэгов и атрибутов
  • Layout rules — стили макета, глобальных элементов шапки, футера, сайдбара и т.п., здесь id в селекторах, т.к. эти элементы встречаются 1 раз на странице
  • Modules rules — стили модулей, то есть блоков, которые могут использоваться несколько раз на одной странице, здесь нет id и селекторов тэгов
  • State rules — стили состояния
  • Theme rules — стили оформлений, которые со временем, возможно, нужно будет заменить (например, новогоднее оформление или для продажи макета)

Multilayer CSS от "Одноклассников"

operatino.github.io/MCSS/, презентация — rhr.me/pres/mcss/

Multilayer CSS от "Одноклассников"

  • Слой фундамента задает нейтральные стили и не влияет на другие слои. Тут может быть любой фреймфорк
  • Элементы базового слоя могут влиять только на классы своего же слоя. Пример: иконки на сайте размером 25x25, но иконки в кнопках — 16x16
  • Элементы проектного слоя могут влиять на базовый и проектный слой. Примеры: иконки в форме логина имеют особый размер 20x20, но модуль «Покупка» имеет форму логина, стили которой модифицированы
  • Косметический слой — в виде OOCSS-классов («атомарные» классы) и не влияет на другой CSS-код, избирательно применяясь в разметке

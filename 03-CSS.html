История CSS

Появление CSS

Термин «каскадные таблицы стилей» был предложен Хокон Виум Ли в 1994 году

В середине 1990-х Консорциум Всемирной паутины стал проявлять интерес к CSS

W3C и CSS

  1. CSS уровень 1 (1996, 1999) — параметры шрифтов, цвета, ...
  2. CSS уровень 2 (12 мая 1998), CSS 2.1 (07 июня 2011) — блочная вёрстка, селекторы, ...
  3. CSS уровня 3 находится в стадии разработки — трансформации, анимация, ...
  4. CSS уровня 4 разрабатывается с 29 сентября 2011 года

Директивы CSS

  1. @charset — Задает кодировку символов
  2. @font-face — Задает описания шрифтов
  3. @import — Включает другую таблицу стилей в текущую
  4. @media — Задает имена устройств отображения
  5. @page — Задает свойства страницы для печати

Типы устройств отображения

Для каждого свойства CSS определяет те устройства отображения, к которым это свойство применимо

  • визуальные, звуковые и тактильные
  • непрерывные и страничные
  • символьные и графические
  • интерактивные и статические
  • all — все устройства

Пример

		@media all {
			BODY { font-size: 10pt }
		}
		@media screen {
  			BODY { font-size: 12pt }
		}

			BODY { line-height: 1.2 }
		}

Типы данных CSS

Целые (100500, -21) и действительные числа (1.25, -.999)

Размеры абсолютные

  • in дюймы (1 дюйм = 2.54 см = 25.4 мм = 72 точки = 6 пик)
  • cm сантиметры (1 см = 10 мм = 0.39 дюйма = 2.36 пики = 28.35 точки)
  • mm миллиметры (1 мм = 0.1 см = 0.039 дюйма = 0.24 пики = 2.84 точки
  • pt точки (1 точка = 1/12 пики = 1/72 дюйма = 0.035 см = 0.35 мм)
  • pc пики (1 пика = 12 точек = 1/6 дюйма = 0.423 см = 4.23 мм)

Типы данных CSS

Размеры относительные

  • em размер (font-size) соответствующего шрифта
  • ex высота строчных букв (x-height) соответствующего шрифта
  • px пиксели (размер зависит от устройства отображения)

Селекторы

Базовые селекторы

  • * — универсальный селектор, все элементы документа
  • селекторы элементов (h1) — все элементы с данным именем
  • простые селекторы — селектор элемента (и *) уточняется селекторами атрибутов, идентификаторов или классов (в любом порядке)
  • сложные селекторы — простые селекторы + иерархия в документе (дети, потомки, соседи)

Оптимизация

		H1 { font-family: sans-serif }
		H2 { font-family: sans-serif }
		H3 { font-family: sans-serif }

эквиваленты одному правилу

H1, H2, H3 { font-family: sans-serif }

Селекторы классов

<p class="warning"> один </p>

<p class="warning red"> два </p>

<p class="warning blue"> три </p>

В таблице стилей:

		p.warning { font-style: italic }
		*.red { color: red }
		.warning.red { font-weight: bold }

Селекторы идентификаторов

<div id="navigation"> ... </div>

В таблице стилей:

		@media print { #navigation { display: none  } }

Обычно идентификатор уникален :)

Селекторы атрибутов

<a href="logo.png" target="_blank"> ... </a>

В таблице стилей:

  • [href] — есть атрибут? хорошо, значение не важно
  • [target="_blank"] — атрибут с конкретным значением
  • [href^="http://"] — начинается с ...
  • [href$=".png"] — или заканчивается ...
  • [href*="github"] — или где-то внутри

Селекторы атрибутов

<p class="block-menu-therm red" lang="en-us"> ...

В таблице стилей:

  • [class~="red"] — одно значение из списка с пробелами
  • [target|="en"] — с начала и до дефиса (напр., для групп языков)

Дочерние элементы наследуют некоторые стилевые свойства своих родителей

Смотрите в спецификацию, интуиция может подвести!

Каскадность

У стилей есть приоритеты в зависимости от источника

  1. Стиль браузера
  2. Стиль автора
  3. Стиль пользователя
  4. Стиль автора с добавлением !important
  5. Стиль пользователя с добавлением !important

сортировка по возрастанию приоритета

Специфичность

  1. подсчитать количество атрибутов id в селекторе
  2. подсчитать количество атрибутов class в селекторе
  3. подсчитать количество названий тегов и псевдоклассов в селекторе (псевдоэлементы игнорируются)

Записать 3 числа подряд, приведя к максимальному основанию

Специфичность

		*              {} /* a=0 b=0 c=0 -> специфичность = 0   */
		li             {} /* a=0 b=0 c=1 -> специфичность = 1   */
		li:first-line  {} /* a=0 b=0 c=2 -> специфичность = 2   */
		ul li          {} /* a=0 b=0 c=2 -> специфичность = 2   */
		ul ol+li       {} /* a=0 b=0 c=3 -> специфичность = 3   */
		ul li.red      {} /* a=0 b=1 c=2 -> специфичность = 12  */
		li.red.level   {} /* a=0 b=2 c=1 -> специфичность = 21  */
		#t34           {} /* a=1 b=0 c=0 -> специфичность = 100 */
		#content #wrap {} /* a=2 b=0 c=0 -> специфичность = 200 */

История HTML

HTML

HyperText Markup Language

  1. Гипертекст — форма письма, которое ветвится или осуществляется по запросу
  2. Язык разметки — набор символов или последовательностей, вставляемых в текст для передачи информации о его выводе или строении

Гипертекст

«Tzar of Science», декан MIT, научный советник президента Рузвельта, идеолог ARPA

1963, Даглас Энгельбарт — «Концептуальная схема усиления человеческого интеллекта»

1965, Тед Нельсон — термин «гипертекст», идея единой литературной среды

История — XML

XML (eXtensible Markup Language), расширяемый язык разметки, предназначен для хранения структурированных данных

XML может содержать любые теги, описанные создателями словаря, но XML не даёт указаний, как должны отображаться данные

  1. XML 1.0 — 10 февраля 1998
  2. XML 1.0 (Fifth edition) — 26 ноября 2008
  3. XML 1.1 (Second edition)— 16 августа 2006

Приложения XML

  1. XHTML — семейство языков разметки веб-страниц на основе XML, повторяющих HTML 4
  2. MathML — язык математических формул
  3. SMIL — язык интеграции и синхронизации мультимедийных средств
  4. SVG — язык двумерной векторной графики

Теги

Тег — элемент языка разметки, заключённый между < и >

  • бывают двойные (открывающий и закрывающий): <div> </div>
  • одиночные (самозакрывающиеся): <input>, в XHTML <input />

<html> <HTML> и <hTml> — одинаковые теги, регистронезависимые (для XML и XHTML это не так)

Теги в HTML можно не закрывать, но так делать не рекомендуется

Вложенность тегов

		<i>очень <b> длинный текст </b> очень</i>  — Правильно
		<i>очень <b> длинный текст </i> очень</b>  — Неправильно
		<!--Браузер это поймет так:-->
		<i>очень <b> длинный текст </b></i> очень</b>

Атрибуты тегов

		<div title="he's alive"> — в двойных кавычках
		<div title='size is 10" more'> — в апострофах
		<div class=footers> — без кавычек (если нет пробелов)
		<input checked> — одиночные атрибуты
		в XHTML значение обязательно <input checked='checked'>

Гиперссылки — AREA и MAP

<map> — описание клиентской карты ссылок

<area> — активные области изображения, которые являются ссылками

Атрибут shape принимает значения

  • rect — прямоугольник, coords="xлевый, yверхний, xправый, yнижний"
  • circle — круг, coords="xцентра, yцентра, радиус"
  • poly — многоугольник, coords="x1, y1, ..., xn, yn"

Валидность

Well-formed (синтаксически корректный)

  1. ровно один корневой элемент
  2. открытые теги обязаны быть закрыты (для XML)
  3. теги не нарушают вложенность
  4. атрибуты тегов заключены в кавычки " (для XML)
  5. символы <, >, & и кавычки экранированы
  6. все символы в документе соответствуют заявленной кодировке

Valid (соответствующий правилам)

  1. правильный корневой элемент
  2. элементы, допустимые внутри какого-то элемента
  3. допустимость текста внутри элемента
  4. обязательность присутствия элемента
  5. обязательность присутствия конкретных атрибутов элемента
  6. расшифровки строковых сущностей ( &rarr; = &#8594; = → )
